- 팀: Link
- 참여자: 윤상이(팀장), 현길용, 최서윤

---

## 1. 공통 헤더 로고 관련 논의

- 헤더 부분에 공통적으로 들어갈 스타일 적용 전,
  **팀 명을 활용한 로고를 먼저 작성하여 적용해보자**는 의견 제시

- **최서윤**
  - 다른 사이트처럼 팀 로고를 클릭하면 메인 화면으로 이동하도록 하자 → O

---

## 2. 공통 레이아웃 적용 방식 논의

공통적으로 사용되는 헤더, 사이드바, 푸터를 어떻게 적용할지에 대한 논의 진행

- **윤상이**
  - 공통 파일을 생성하여 모든 페이지에 적용하자

- **현길용**
  - 페이지마다 필요한 HTML 구조가 다르므로
    공통 파일보다는 각각 따로 작성하는 것이 좋을 것 같다는 의견

### 결정 사항
- 아직 **미정**

## 6. 회고록

- **윤상이**
  - GitHub에 대한 지식 부족으로 작성된 파일들을 삭제하는 문제가 발생하여 현재 재작업 중
  - 프로젝트 진행 중 파일 접근 방식과 보관 방법의 중요성,
    그리고 데이터 안전성에 대해 절실히 깨달음
  - 이후부터는 일정 단위(약 1 ~ 2개 작업 완료 시)마다
    반드시 백업을 진행하는 방식으로 변경함

- **현길용**
  - 역할 분담이 명확하여 작업 흐름이 끊기지 않았음
  - 중간에 막히는 부분은 팀장님의 도움으로 수월하게 해결할 수 있었음
  - GitHub 사용 시 유의해야 할 점이 많다고 느낌

- **최서윤**
  - 배치를 수정하는 과정에서 적용이 잘 되지 않는 문제가 있었으나,
    팀원들의 도움으로 빠르게 해결할 수 있었음
  - 협업의 긍정적인 효과를 체감한 작업이었다
